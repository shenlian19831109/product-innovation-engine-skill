# 需求挖掘与产品创新引擎 (v9.0)

本 Skill 提供了一套从场景输入到产品创新方案的全流程操作指南，旨在将碎片化的用户表达转化为高价值的产品洞察。

## 核心方法论

1.  **场景细分法**：将宽泛场景拆解为具体的子场景。
2.  **三层挖掘法**：需求点 → 痛点 → 爽点。
3.  **9 大需求理论**：利用马斯洛、双因素、成就动机等理论进行深度诊断。
4.  **为什么×5**：从痛点逐层深挖到根本原因与深层爽点。
5.  **1-6 层价值阶梯**：从基础功能（生存）到颠覆功能（品牌）。

## 执行流程

### 1. 场景细分
将用户输入的行业场景（如：电商、教育、办公、内容创作、医疗、金融、制造）拆解为具体的流程环节。
- **操作**：识别行业流程，补充特有场景，请用户确认聚焦的子场景。

### 2. 数据采集与整理
从多渠道（小红书、抖音、知乎、Amazon、TikTok 等）采集用户抱怨、求助、求推荐及竞品差评。
- **工具建议**：MediaCrawler (国内), Bright Data (海外), Firecrawl (网页), OpenClaw (AI 辅助)。
- **整理**：去重、去噪、分类（内容类型、需求类型、用户类型、情绪强度）。

### 3. 趋势与强度分析
- **趋势**：计算长尾词增长率，识别爆发（>100%）或快速增长（30%-100%）的需求。
- **强度评分**：需求强度 = 频次分×0.3 + 情绪分×0.2 + 增长分×0.2 + 付费意愿分×0.2 + 市场规模分×0.1。

### 4. 需求深度挖掘
- **三层挖掘**：识别用户直接说的（需求点）、隐含的不满（痛点）和没想到的惊喜（爽点）。
- **理论诊断**：应用马斯洛需求层次、双因素理论等。
- **为什么×5**：通过连续追问找到根本原因，定义核心价值。

### 5. 产品创新输出
- **功能分层**：
    - 1-2 层：生存（基础/改进功能）
    - 3-4 层：赚钱（深化/洞察功能）
    - 5-6 层：品牌（惊喜/颠覆功能）
- **分阶段策略**：规划 MVP（验证需求）、增长（建立差异化）、成熟（形成壁垒）三个阶段的定价与营销话术。
